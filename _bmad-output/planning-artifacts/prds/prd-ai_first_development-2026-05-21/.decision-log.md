# Decision Log — International Roaming Intelligence PRD

## Decisions

### D-1: Custom web app over BI tool (2026-05-21)
**Decision:** Build a purpose-built web application on Cloud Run (bi-layer) rather than using Looker/Looker Studio or another BI tool.
**Rationale:** User preference. More flexibility for the steering team's specific workflow, though more engineering effort.
**Status:** Confirmed by user.

### D-2: Data quality monitoring in V1 (2026-05-21)
**Decision:** Include data quality checks (freshness, coverage, volume anomaly detection) in the MVP scope.
**Rationale:** User preference. The steering team needs confidence in data trustworthiness before acting on it.
**Status:** Confirmed by user.

### D-3: Internal tool stakes calibration (2026-05-21)
**Decision:** PRD scoped at internal-tool depth (~5-8 pages). Single team, no external users, no revenue-facing surface.
**Rationale:** Appropriate rigor for the audience and risk profile.
**Status:** Confirmed by user.

### D-4: Fast path working mode (2026-05-21)
**Decision:** Draft full PRD with [ASSUMPTION] tags, then iterate on user review.
**Rationale:** Solid product brief provides sufficient context for inference.
**Status:** Confirmed by user.

### D-5: Composite Quality Score equal-weight default (2026-05-21)
**Decision:** V1 uses equal weight across all available KPIs (each normalized 0–100, then averaged). Steering team refines post-launch.
**Rationale:** Reviewer finding — score was a black box blocking downstream work. Equal-weight is a reasonable unbiased default; refinement requires operational experience.
**Status:** Applied in PRD.

### D-6: KPI degradation direction specification (2026-05-21)
**Decision:** FR-12 specifies per-KPI degradation direction (Latency/Packet Loss: increase = worse; Throughput/Session Success Rate: decrease = worse).
**Rationale:** Reviewer finding — "degradation" was ambiguous without direction.
**Status:** Applied in PRD.

### D-7: PRD finalized (2026-05-21)
**Decision:** PRD set to `status: final` after reviewer pass and targeted fixes.
**Rationale:** User approved. Reviewer verdict: adequate — ready for handoff. Critical/high findings resolved.
**Status:** Final.

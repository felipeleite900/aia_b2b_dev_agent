# Architecture Pivot: Static HTML Reports

The PRD's Feature 4.7 (deployed web application with Cloud Run, IAP, real-time interactivity) is replaced by **static HTML reports generated by the pipeline**. This eliminates the web application subsystem entirely and consolidates the project into a single subsystem: a BigQuery data pipeline that produces shareable HTML output.

**Rationale:** The steering team's workflow is better served by a daily HTML report they can open, share, and review offline. No deployment infrastructure, no authentication layer, no ongoing hosting. Distribution is manual (file sharing).

**Impact on PRD requirements:**
- FR-16 (authenticated access): Replaced by file-level access control (who receives the file)
- FR-17 (responsive data views): Replaced by Plotly's built-in chart interactivity (hover, zoom, filter)
- FR-18 (filter and search): Simplified to Plotly's native filtering + HTML-embedded table sorting
- FR-4 through FR-12 (quality views, rankings, trends): Delivered as sections within the HTML report
- All pipeline FRs (FR-1 to FR-3, FR-13 to FR-15): Unchanged
